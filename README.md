## About

Strict ESLint presets for modern TypeScript projects.

## Features

- Enforces type safety
- Prohibits potentially unsafe patterns
- Encourages modern syntax
- Encourages the use of arrow functions
- Encourages the use of try/catch
- Bans `var` and warns about proper use of `const` and `let`
- Disallows interfaces, using only types
- Requires explicit readonly and accessibility modifiers in classes
- ...and much more!

## Installation

You can install it as follows.

> [!NOTE]
> Along with this, the following packages are installed automatically: @eslint/js, @types/node, eslint, jiti, typescript, and typescript-eslint. This is done to ensure that Neatlint remains stable, up to date, and works without errors. Therefore, when you install Neatlint, you do not need to install these packages manually. If you have previously installed them, you also do not need to keep them in your dependencies.

```shell
# NPM
npm add neatlint

# PNPM
pnpm add neatlint

# Yarn
yarn add neatlint

# Bun
bun add neatlint

# Deno
deno add neatlint
```
